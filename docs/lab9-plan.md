# Lab 9 Plan

## Product Idea

- Product title: Happiness Menu
- End user: Innopolis University students, staff, and campus visitors
- Problem: Users waste time in line while trying to understand the cafe menu
- One-line idea: A bilingual web app for browsing the Happiness Cafe menu and updating it through a lightweight staff panel
- Core feature: Fast browsing of the menu by section and category before reaching the counter

## Version 1

Version 1 does one thing well:

- Public web menu
- Browsing by section
- Browsing by category
- Russian and English switch
- PostgreSQL-backed backend API
- Lightweight menu assistant agent for recommendations

## Version 2

Version 2 extends the product:

- Staff login
- Owner and worker roles
- Add, update, and delete menu items
- Owner can add and remove workers
- Improved assistant that uses live menu data from the database
- Dockerized deployment
